# 프로젝트 상태 및 명세서

## 1. 프로젝트 개요
- **목표**: 운동 기록 및 AI 기반 맞춤형 운동 플랜 추천 웹 애플리케이션 (PWA 지향)
- **주요 기능**:
    - 사용자 신체 정보 기반 운동 목표 설정 (온보딩)
    - AI 운동 분할 및 스케줄 추천
    - 운동 기록 및 캘린더 연동 시각화
    - 운동 성과 분석 및 동기 부여 메시지

## 2. 현재 구현 상태 (2026-01-31 기준)

### 2.1 인증 및 온보딩
- **로그인**: Supabase Auth 연동 완료
- **온보딩 플로우**:
    - 키, 현재 체중, 목표 체중, 현재/목표 골격근량 입력
    - `ScrollPicker` UI 컴포넌트 적용 (모바일 친화적 UX)
    - **Issue**: 마지막 단계에서 'AI 플랜 생성하기' 클릭 시 로딩 화면(Step 7)으로 전환되지 않는 버그 확인됨 (수정 예정).

### 2.2 메인 대시보드
- **구조**: `app/(main)/dashboard/page.tsx` 및 `dashboard-view.tsx`
- **기능**:
    - 사용자 목표, 운동 계획(Plan), 기록(History), 최근 운동 로그 표시
    - 데이터가 없으면 온보딩으로 리다이렉트 처리

### 2.3 기술 스택
- **Framework**: Next.js 16.1.4 (App Router)
- **Database**: Supabase (PostgreSQL)
- **Styling**: Tailwind CSS, Shadcn UI, Framer Motion
- **State/Form**: React Hook Form, Zod

## 3. 남은 작업 및 개선 사항

### 3.1 AI 운동 추천 시스템 고도화
- 단순 목업/기본 로직에서 벗어나, 입력된 신체 정보를 바탕으로 실제 운동 분할(2분할, 3분할 등) 추천 로직 구현 필요.

### 3.2 캘린더 및 기록 시각화
- 홈 화면 달력 컴포넌트에 운동 일정 노출.
- 과거 운동 기록 클릭 시 상세 정보 모달/페이지 이동.
- D-Day 표시 및 동기 부여 메시지 표시.

### 3.3 운동 실행 및 완료 UX
- 운동 중 세트/무게/횟수 입력 인터페이스 개선.
- 운동 종료 후 AI 분석 코멘트 표시.

## 4. 참고 사항
- `.gemini` 폴더는 에이전트 간 컨텍스트 공유를 위해 사용됨.
- UI/UX는 모바일 퍼스트로 설계.
